es 是基于lucene开发出的近实时文档型数据库

看到第六章【一本书讲透elasticsearch】
分片
分片：索引数据被分片存在不同的节点上，分片本质是一个Lucene索引
副本：对副分片的复制分片称为副本分片

节点：

  1. 主节点：负责集群的管理和源数据的维护，索引的创建和删除操作，确保集群正常运行，不需要涉及到文档级别的变更和搜索等操作
  2. 数据节点: 存储，检索和处理数据，提供搜索和聚合功能（数据节点，热数据节点，冷数据节点，温数据节点）
  3. 协调节点：处理客户端请求，协调数据节点工作，优化分布式搜索
  4. ingest节点：预处理节点，负责数据预处理，如过滤，转换等

映射类型

  1. 动态映射：不创建index，直接写入，boolean，float，long，(嵌套数据类型)object，array，date，字符串类型能被检测，其他的会映射成text，可能会出现字段不匹配的情况
  2. 静态映射：需要先建立索引，明确字段类型
    1. dynamic : false, 可以不创建索引写入数据，但是不能被_search, 可以被_get
    2. dynamic : strict, 写入报错
      如果不希望doc被索引，可以在mappings的设置中将enabled:false,如果只是想索引enabled:false文档的其他字段，可以将该字段的strore:true

null_value
默认情况下null字段不能被检索，可以设置null_value:"NULL"改为可null检索，支持的字段有arrays，boolean，date，Gore—point，ip, keyword, numeric, point等

数据类型

数据类型 类型类别 描述
text 核心数据类型 用于全文搜索的文本数据，会被分析（分词）。
keyword 核心数据类型 用于精确值搜索，不会被分析（不分词），适用于过滤、排序和聚合操作。
date 核心数据类型 日期或日期时间，支持多种日期格式。
long 数值类型 用于长整型数据。
integer 数值类型 用于整型数据。
short 数值类型 用于短整型数据。
byte 数值类型 用于字节数据。
double 数值类型 用于双精度浮点数数据。
float 数值类型 用于浮点数数据。
boolean 核心数据类型 布尔类型，接受true和false。
object 复杂数据类型 用于单个JSON对象。
nested 复杂数据类型 用于对象数组，支持数组中的每个对象独立索引和查询。
geo_point 地理位置数据类型 用于地理位置数据，支持经纬度坐标。
geo_shape 地理位置数据类型 用于复杂地理位置数据，如多边形等。
ip 特殊数据类型 用于IP地址，支持IPv4和IPv6。
completion 特殊数据类型 用于自动补全功能，快速提供搜索建议。
token_count 特殊数据类型 用于统计分词后的词元数量。
integer_range 范围数据类型 用于存储整数的范围数据。
float_range 范围数据类型 用于存储浮点数的范围数据。
long_range 范围数据类型 用于存储长整型的范围数据。
double_range 范围数据类型 用于存储双精度浮点数的范围数据。
date_range 范围数据类型 用于存储日期的范围数据。

与Mysql的对比

以下表格展示了Elasticsearch和MySQL在不同概念上的映射关系，帮助理解它们在概念上的对应和差异：

Elasticsearch MySQL 描述
索引(Index) 数据库(Database) 存储数据的顶层单位。Elasticsearch的索引是数据的容器，MySQL的数据库也是。
类型(Type)（已废弃） 表(Table) 以前Elasticsearch使用类型来区分索引内的数据，但现在已经废弃。MySQL的表是结构化数据的集合。
文档(Document) 行(Row) 数据的基本单位。Elasticsearch存储JSON文档，而MySQL存储行数据。
字段(Field) 列(Column) 数据的属性。Elasticsearch的文档由字段组成，MySQL的行由列组成。
映射(Mapping) 表结构(Schema) 定义数据结构的规则。Elasticsearch的映射定义了索引中文档的字段类型，MySQL的表结构定义了列的数据类型。
倒排索引(Inverted Index) 索引(Index) 优化搜索的数据结构。Elasticsearch通过倒排索引实现快速文本搜索，MySQL的索引（如B树索引）优化查询性能。
查询DSL(Query DSL) SQL(Structured Query Language) 查询语言。Elasticsearch使用Query DSL进行数据查询，MySQL使用SQL。
分片(Shard) 分区(Partition) 数据分布策略。Elasticsearch用分片实现数据的水平分割和分布，MySQL的分区也是将数据分散存储。
复制(Replica) 副本(Replica) 数据的冗余副本。Elasticsearch和MySQL都支持数据复制，提高数据的可用性和读取性能。

Term查询

Term查询对查询内容不做分词，直接去倒排索引里去匹配文档，精确匹配。

组合查询

复杂查询基本会用到bool关键字。

  • must : 多个查询条件的完全匹配,相当于 and。
  • must_not ::多个查询条件的相反匹配，相当于 not。
  • should : 至少有一个查询条件匹配, 相当于 or

boosting query(提高查询)

不同于bool查询，bool查询中只要一个子查询条件不匹配那么搜索的数据就不会出现。而boosting query则是降低显示的权重/优先级（即score)。

比如搜索逻辑是 name = ‘apple’ and type =‘fruit’，对于只满足部分条件的数据，不是不显示，而是降低显示的优先级（即score)

query_string查询

query_string查询在Elasticsearch中提供了强大且灵活的查询语法，允许用户执行复杂的搜索，包括字段特定搜索、通配符搜索、布尔搜索等。这种查询类型让用户可以在一个查询中结合使用多种查询语法，非常适合那些需要执行高级文本搜索的场景。
