203 移除链表元素

【题目描述】

删除链表中等于给定值 val 的所有节点，并返回新的头节点。

【解题思路】

解决这个问题的关键是要注意我们可能需要删除头节点，因为头节点也可能等于给定的值 val。因此，一个常见的方法是添加一个哑节点（dummy node）作为新的头节点，这样可以统一处理头节点和其他节点的删除操作。

  1. 添加哑节点：在原链表的头节点之前添加一个哑节点，哑节点的 next 指向原链表的头节点。
  2. 遍历链表：遍历链表，比较当前节点的下一个节点的值是否等于给定值 val。如果等于，就需要删除这个节点。这可以通过改变当前节点的 next 指针来实现，让它指向下下个节点。
  3. 返回结果：遍历完成后，返回哑节点的 next 节点，即新的头节点。

707 设计链表

【题目描述】

设计链表的实现。您可以选择使用单链表或双链表。单链表中的节点应该具有两个属性：val 和 next。val 是当前节点的值，next 是指向下一个节点的指针/引用。如果要使用双向链表，则还需要一个属性 prev 以指示链表中的上一个节点。在链表类中实现这些功能：

  • get(index)：获取链表中第 index 个节点的值。如果索引无效，则返回-1。
  • addAtHead(val)：在链表的第一个元素之前添加一个值为 val 的节点。插入后，新节点将成为链表的第一个节点。
  • addAtTail(val)：将值为 val 的节点追加到链表的最后一个元素。
  • addAtIndex(index, val)：在链表中的第 index 个节点之前添加值为 val 的节点。如果 index 等于链表的长度，则该节点将附加到链表的末尾。如果 index 大于链表长度，则不会插入节点。如果 index 小于0，则在头部插入节点。
  • deleteAtIndex(index)：如果索引 index 有效，则删除链表中的第 index 个节点。

【解题思路】

在这个问题中，我们将实现一个单链表，其中包含一个哑节点（dummy node）作为辅助节点，以简化在链表头部添加和删除节点的操作。

  1. 初始化：链表初始化时，创建一个哑节点，哑节点的 next 属性指向 null。维护一个 size 变量来跟踪链表的长度。
  2. 获取节点值：在获取第 index 个节点的值时，首先检查 index 是否有效（即 0 <= index < size），然后从哑节点开始遍历到第 index 个节点，返回其值。
  3. 在头部添加节点：创建新节点，将新节点的 next 指向原头节点，哑节点的 next 指向新节点，链表长度 size 加1。
  4. 在尾部添加节点：遍历到链表的最后一个节点，将其 next 指向新节点，链表长度 size 加1。
  5. 在指定位置添加节点：首先检查 index 是否有效，如果有效（包括在头部添加和尾部添加的特殊情况），则遍历到指定位置的前一个节点，按照插入逻辑添加新节点。
  6. 删除节点：如果 index 有效，遍历到指定位置的前一个节点，改变前一个节点的 next，以跳过需要删除的节点。

24 两两交换链表中的节点

【题目描述】

给你一个链表，两两交换其中相邻的节点，并返回交换后的链表。你不能只是单纯的改变节点内部的值，而是需要实际的进行节点交换。

【解题思路】

解决这个问题的关键是理解两两交换链表中节点的过程。我们可以使用迭代的方式来完成这个任务。对于链表中的两个相邻节点，我们可以通过更新节点指针的指向来交换这两个节点。

具体步骤如下：

  1. **初始化一个哨兵节点(dummy node)**，这个节点的下一个指向链表的头节点。这样做的目的是为了简化在链表头部进行操作的逻辑，使得头节点和中间的节点可以用同一套逻辑来处理。
  2. 遍历链表：使用一个指针（我们称之为prev）从哨兵节点开始遍历，每次处理一对节点。prev的下一个节点就是这对节点中的第一个，而prev的下一个节点的下一个就是这对节点中的第二个。
  3. 交换两个节点：改变prev和这对节点的指针关系，使得第二个节点指向第一个节点，第一个节点指向第二个节点的下一个节点，最后prev指向新的第一个节点（即原来的第二个节点）。
  4. **移动prev**：将prev移动到下一对需要交换的节点的前一个位置。
  5. 当链表的长度是奇数时，最后一个节点不需要交换，直接保持即可。

这样，通过一次遍历，我们就可以完成整个链表的两两节点交换。
