双指针

jonas
2024-04-07
leetcode
leetcode, 双指针

双指针解法通常是使用两个指针同时移动来实现

常见的有：

  1. 快慢指针：移出元素，删除倒数第K个元素，链表环
  2. 首尾指针：反转字符串,N数之和

27 移除元素

【题目描述】

给你一个数组 nums 和一个值 val，你需要原地移除所有数值等于 val 的元素，并返回移除后数组的新长度。不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并原地修改输入数组。元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

【解题思路】

  1. 双指针法：这个问题可以用双指针法高效地解决。定义两个指针：一个快指针（fast）和一个慢指针（slow）。快指针用于遍历数组，慢指针用于跟踪非val元素的位置。
  2. 遍历数组：快指针（fast）从头到尾遍历数组。对于每次遍历：
    • 如果快指针指向的元素不等于val，就将快指针指向的元素复制到慢指针的位置，然后两个指针都前进一步。
    • 如果快指针指向的元素等于val，就只移动快指针，慢指针保持不动。
  3. 返回结果：遍历结束后，慢指针的位置就是数组中非val元素的新长度。

151 翻转字符串里的单词

【题目描述】

给你一个字符串 s ，逐个翻转字符串中的所有 单词 。

单词 是由非空格字符组成的字符串。s 中使用至少一个空格将字符串中的 单词 分隔开。

请你返回一个翻转 s 中单词顺序并用单个空格连接的字符串。

说明：

  • 输入字符串 s 可以在前面、后面或者单词间包含多余的空格。
  • 翻转后的字符串中不应包含额外的空格。

【解题思路】

要在O(1)空间复杂度下翻转字符串里的单词，我们不能使用额外的数组来存储分割或翻转后的结果。因此，需要在原字符串上直接进行操作。解题思路可以分为以下几步：

  1. 去除多余空格：首先，我们需要在原字符串上去除首尾以及单词间的多余空格。这可以通过双指针技巧在原地完成。
  2. 翻转整个字符串：接下来，我们翻转整个字符串。这样做是为了让单词的顺序颠倒过来。
  3. 翻转每个单词：最后，我们再次翻转字符串中的每个单词，这样单词内字符的顺序就恢复正常了，而单词的顺序则是反的。

206 反转链表

【题目描述】

给你单链表的头节点 head ，请你反转链表，并返回反转后的链表。

【解题思路】

反转链表是一个非常基础且常见的链表操作题。解决这个问题，我们主要可以采用迭代或递归的方式。这里，我将介绍迭代的方法，因为它比较直观且容易理解。

迭代方法的核心思想是遍历原链表，将每个节点的 next 指针指向其前一个节点。由于节点没有引用其前一个节点，我们需要额外存储前一个节点。同时，为了保持遍历的进行，我们也需要存储当前节点的下一个节点（因为修改 next 指针后，会丢失原本的下一个节点）。

具体步骤如下：

  1. 初始化两个指针 prev 和 curr。prev 初始化为 nil，curr 初始化为头节点 head，表示当前遍历到的节点。
  2. 遍历链表直到 curr 为 nil。在遍历过程中，先暂存 curr 的下一个节点（因为后面需要修改 curr 的 next），然后将 curr 的 next 指向 prev，以此来反转链表。之后，prev 和 curr 都前进一步，继续遍历和反转。
  3. 当遍历结束时，prev 将会是新的头节点。

19 删除链表的倒数第N个节点

【题目描述】

给你一个链表，删除链表的倒数第 n 个节点，并且返回链表的头节点。

示例：

【解题思路】

要删除链表的倒数第 n 个节点，一个直观的思路是先遍历一遍链表以确定链表的总长度 L，然后再次遍历到第 L-n 个节点并进行删除操作。但是，这需要两次遍历，是否有方法只遍历一次就解决呢？

这里我们可以使用双指针技术中的“快慢指针”方法。具体步骤如下：

  1. 创建一个哑结点（dummy node），它的 next 指针指向链表的头节点。这是为了便于处理边界情况，比如链表只有一个节点或需要删除头节点的情况。
  2. 初始化两个指针 fast 和 slow，它们都指向哑结点。
  3. 先移动 fast 指针，使其与 slow 指针之间相隔 n 个节点。
  4. 然后同时移动 fast 和 slow 指针，直到 fast 指向链表的末尾。
  5. 此时 slow 指针的下一个节点就是要删除的节点。将 slow.next 指向 slow.next.next，即可完成删除操作。
  6. 返回哑结点的下一个节点，即为新的头节点。

160 相交链表

【题目描述】

编写一个程序，找到两个单链表相交的起始节点。如果两个链表没有交点，返回 null。

为了保证性能，你的代码应满足 O(n) 时间复杂度和 O(1) 空间复杂度。链表中节点的数量范围是 [0, 10^9]。链表节点的值的范围是 [-10^9, 10^9]。

注意：

  • 函数返回结果后，链表必须保持其原始结构。
  • 可假定整个链表结构中没有循环。
  • 程序尽可能在 O(n) 时间复杂度和 O(1) 空间复杂度内运行。

【解题思路】

要找到两个链表相交的起始节点，一种直观的方法是使用两个指针分别遍历两个链表。当一个指针到达链表末尾时，将它移动到另一个链表的头部继续遍历；同样，另一个指针也这样处理。如果两个链表相交，那么这两个指针最终会在相交节点相遇。

这个方法之所以有效，是因为两个指针分别遍历两个链表后，路径长度相同（两个链表的长度和再加上它们的公共部分），所以它们会同时到达相交节点。

142 环形链表 II

【题目描述】

给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意，pos 仅仅是用于标识环的参数，你并不需要对链表进行修改。

示例：

【解题思路】

解决这个问题的关键是使用快慢指针的方法。快指针每次移动两步，慢指针每次移动一步。如果链表中存在环，则快慢指针最终会在环内相遇。之后，我们可以用以下步骤找到环的入口：

  1. 当快慢指针首次相遇时，将快指针重新定位到链表的头部，并将其移动速度改为每次一步。
  2. 然后，快慢指针再次同时出发，每次都移动一步。
  3. 当它们再次相遇时，相遇点就是环的入口。

这个方法的正确性基于数学证明，关键在于理解，当快慢指针首次在环内相遇时，慢指针没有遍历完整个链表，而此时快指针已经在环内多转了若干圈。无论环的大小，将快指针重置并改为每次移动一步，它们都会在环的入口相遇。

15 三数之和

【题目描述】

给你一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a，b，c，使得 a + b + c = 0？请你找出所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

示例：

【解题思路】

解决这个问题的关键是如何有效地处理重复元素和如何将三数之和问题降为两数之和问题。

  1. 排序：首先对数组进行排序，这一步是为了后续去重和降低问题复杂度做准备。
  2. 遍历+双指针：遍历排序后的数组，对于每个元素，使用双指针在当前元素之后的范围内寻找两个数使它们的和加上当前元素的和为0。具体做法是：固定一个数 nums[i]，然后使用左右指针（left=i+1，right=len(nums)-1）在 nums[i] 之后寻找 nums[left] + nums[right] = -nums[i]。
  3. 去重：在遍历和双指针移动过程中，需要小心地去除重复的三元组。这可以通过跳过数组中连续相同的元素来实现。
